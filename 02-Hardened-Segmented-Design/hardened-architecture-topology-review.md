# Hardened Architecture: Topology Review
**Explanation of the [Hardened Architecture Diagram](https://github.com/Herman-Samuel/homelab-infrastructure/blob/main/02-Hardened-Segmented-Design/hardened-architecture-topology-review.md)**

This document provides a technical analysis of the hardened network architecture diagram. The transition from the legacy flat model to this segmented model utilizes 802.1Q VLAN tagging and firewall rules to enforce isolation between different segments. 

---
### **Diagram Flow**
- At the top is the title, "Hardened Architecture: Segmented Network Model", which clearly emphasizes that this diagram is about the more secure segmented network that I am designing and implementing.
- The start of the path is an internet symbol, representing the untrusted public web.
- An arrow goes from the internet symbol to the ISP Gateway titled "Edge Gateway". Under it is a note stating that it is set to IP Passthrough, meaning all this device is doing is acting as the "courtyard door", passing the public IP Address to my custom router.
- Next, an arrow goes from the edge gateway to a light pink box titled "VLAN 60: Management", and has 10.0.60.0/24 as its private IP Address range. "Out-of-Band Management" is written in green here to indicate a fix from the Legacy Architecture Diagram which has "In-Band Management". On the right hand side of the Management VLAN box is a green shield with a checkmark and green text that reads "Enterprise-Grade Perimeter & Inter-VLAN Security". This is a fix from the Legacy Architecture Diagrams red text "Consumer-Grade Perimeter Security".
- The arrow from the edge gateway continues to inside the light pink VLAN 60 box, pointing towards a darker pink box residing within, titled "OPNsense Router/Firewall". This is my custom built and configured router and firewall, which is able to handle inter-VLAN routing and advanced security (e.g. Intrusion Prevention System and Next Generation Firewall).
- From the OPNsense router the arrow points to a managed switch. This managed switch is what the access point and wired devices are plugged into. Each port is configured to a "trunk port" for VLAN tagging or an "access port" for direct device plug in to a specific VLAN.
- Port number assignments:
	- Port 1 (Trunk Port): OPNsense Router
	- Ports 2-4 (Access Ports): VLAN 10- Main
	- Ports 5-6 (Access Ports): VLAN 30- Server
	- Port 7 (Access Port): VLAN 50- Sandbox
	- Port 8 (Trunk Port): Access Point
- From the managed switch there is an arrow going VLANs 10, 30, 50, and the Access Point. These represent the wired connections from the switch.
- From the Access Point an arrow points to VLANs 10, 20, and 40. This represents Wi-Fi access.
- Further information on each VLAN in the diagram can be seen in the next two sections.
- From VLAN 10: Main, there is a green arrow to all of the other VLANs and to the internet, representing the Main VLANs "Full Access". Devices in the Main VLAN initiate stateful sessions to "start conversations" with any other segment, allowing that VLAN to talk back for just that specific purpose and instance.
- From VLAN 20: Guest, there is a green arrow going to the internet, representing normal internet access. There are no other arrows coming from VLAN 20 because it does not have access to the other VLANs.
- From VLAN 30: Server, there is a yellow arrow going to the Main VLAN, representing restricted access for initiating the sending of data, mostly security & network monitoring notifications. There are no other arrows coming from VLAN 30 because, besides its restricted push to Main, it does not have access to the other VLANs.
- From VLAN 40: IoT, there is a yellow arrow going to the internet, representing restricted access to the internet. The IoT VLAN is restricted to using internet access only with the specific ports necessary to run the services the devices on it require. There are no other arrows coming from VLAN 40 because it does not have access to the other VLANs.
- From VLAN 50: Sandbox, there is a yellow arrow going to the internet, representing restricted access to the internet. The Sandbox only connects to the internet through a heavily monitored and restricted proxy running on the OPNsense router. There are no other arrows coming from VLAN 50 because it not have access to the other VLANs.
- From VLAN 60: Management, there is a white arrow going to the VLANs and Access Point connected to the switch, and a white arrow going to the VLANs connected to the Access Point. This represents the architecture of how the VLANs are connected. There are no other arrows coming from VLAN 60 because it is inbound from Main only. Besides the forwarding of packets that happens on the network devices, the Management VLAN does not have access to the other VLANs.
- There is a "Legend" near the bottom in the center of the diagram with some information
	- "Restricted Lateral Movement" is in green text, stating that inter-VLAN traffic is controlled by the OPNsense router. This is a fix from the Legacy Architectures red text "Unrestricted Lateral Movement".
	- "Default Deny" is in bold representing the firewalls base state. The firewall will deny any traffic that is not explicitly allowed, that way lateral movement is restricted by default and I can fully control what traffic is allowed where.
	- There is a green arrow equals "Allowed" definition
	- There is a yellow arrow equals "Allowed with restrictions" definition
- At the very bottom center of the diagram is text that reads "Segmented Broadcast Domain (10.0.0.0/16)". This is a change from the Legacy Architecture Diagram that was a single broadcast domain. The 10.0.0.0/16 represents the private IP Address Range of the network. The /16 represents the subnet mask 255.255.0.0 (first two octets are the network portions, last two octets are the host portions).
	- Under this is "Separate DHCP Pools" in green text, representing the fix from the legacy diagram of DHCP for all devices coming from the same "pool". Each VLAN has a subnet mask 255.255.255.0, represented by the /24 notation, each with their own unique network portion in the third octet. The first three octets are the network portions, last octet is the host portion. This means that DHCP can't assign a device in the IoT segment an IP Address that belongs to a device in the Main segment.
- In the bottom left corner is in smaller text is a note that the green text represents the fixed red text from the Legacy Architecture Diagram.

### **The Management Plane (VLAN 60)**
- **Core Infrastructure:** The OPNsense router, Managed Switch, and Wireless Access Point reside in this isolated segment.
- **Control logic:** Management interfaces are no longer accessible from general-purpose segments (like Guest or IoT). Administrative access is restricted specifically to authorized devices in the Main VLAN, protecting the "brain" of the network from credential sniffing or unauthorized configuration changes.
- The 10.0.60.0/24 represents the private IP Address Range of the VLAN.
---
### **VLAN Segmentation & Trust Levels**
The network is partitioned into five client segments, each with a specific security policy:
- **VLAN 10: Main (Trusted):** High-trust workstations and personal devices. This segment has stateful access to all other VLANs for management and printing, but is protected from incoming unsolicited traffic from lower-trust zones. The 10.0.10.0/24 represents the private IP Address Range of the VLAN. There is also a note that this VLAN has wired and wireless access.
- **VLAN 20: Guest (Untrusted):** Wireless-only segment with a strict "WAN-Only" policy. Devices can reach the Internet but have zero visibility into the internal 10.0.0.0/16 private space. The 10.0.20.0/24 represents the private IP Address Range of the VLAN. There is also a note that this VLAN has wireless access only.
- **VLAN 30: Server (Internal Services):** The homes static infrastructure like the NAS, Printer, and the Raspberry Pi (Security Stack). Traffic is restricted to specific ports (e.g., 514 for Syslog, 9997 for Splunk) to ensure the server segment remains a secure repository for data and telemetry. The 10.0.30.0/24 represents the private IP Address Range of the VLAN. There is also a note that this VLAN has wired access only.
- **VLAN 40: IoT (Restricted):** Contains high-risk "Smart Home" devices and cameras. These are blocked from the LAN and limited to specific external domains (e.g., Ring/Microsoft) to prevent phone-home telemetry to unauthorized servers. The 10.0.40.0/24 represents the private IP Address Range of the VLAN. There is also a note that this VLAN has wireless access only.
- **VLAN 50: Sandbox (High Risk):** A strictly isolated "Dirty Lab" for malware analysis and high-risk research. Traffic is "Allowed with Restrictions" to the WAN to allow for research while maintaining a total block on all internal communication to prevent lateral spread. The 10.0.50.0/24 represents the private IP Address Range of the VLAN. There is also a note that this VLAN has wired access only.

### **Security Concerns Mitigated (Green Text Analysis)**
This architecture directly remediates the "Red Text" vulnerabilities found in the Legacy Audit:
- **Out-of-Band Management:** Administrative traffic is now physically and logically isolated from general user and IoT traffic via a dedicated Management VLAN (60). Access to the OPNsense WebGUI and SSH console is restricted to this encrypted segment, effectively neutralizing the risk of credential sniffing or lateral movement from compromised household devices.
- **Enterprise-Grade Perimeter:** Replaced the passive ISP Gateway with a Stateful Inspection Firewall (OPNsense) capable of granular East-West traffic filtering. Eliminated "Internal Blindness" by implementing comprehensive logging and monitoring. Unlike the legacy gateway, OPNsense provides real-time traffic flow data and system telemetry, which is forwarded to a Raspberry Pi-based SIEM (Splunk). OPNsense is also capable of advanced security controls like an Intrusion Prevention System (IPS) and Next Generation Firewall (NGFW). This transforms the gateway from a simple "locked door" into an active security monitor that can neutralize malicious activity originating from compromised IoT devices or external WAN actors.
- **Restricted Lateral Movement:** Eliminated the risk of "East-West" traversal by replacing the single broadcast domain with 802.1Q VLAN Segmentation. Each device category (IoT, Main, Guest, etc.) is now isolated within its own separated sub-network. Communication between these zones is governed by strict firewall ACLs (Access Control Lists) that follow a Zero-Trust philosophy. This means that even if a security camera is compromised, the attacker is logically trapped within the IoT VLAN and cannot "walk" over to the Main PC or Management workstation.
- **Separate DHCP Pools:** Eliminated the "Shared Pool" risk by implementing isolated DHCP Scopes for each unique VLAN. By mapping device types to specific subnets (e.g., IoT on 10.0.40.x vs. Main on 10.0.10.x), the OPNsense firewall can now apply 'Identity-Aware' security policies. This ensures that high-security administrative rules are never accidentally applied to low-security appliances, and allows for precise MAC-to-IP reservations to maintain a deterministic, auditable network inventory.

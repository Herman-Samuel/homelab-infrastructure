## **Primary Mission**
To transition from a "Flat Network" (192.168.1.0/24) to a hardened, segmented infrastructure that balances daily productivity with a strictly monitored environment for security research and malware analysis.

#### **Core Technical Objectives**
- **Logical Segmentation (Zero Trust):** Eliminate the single broadcast domain using 802.1Q VLAN tagging. This will logically separate devices by purpose (Main, Guest, Server, IoT, Sandbox, and Management), resulting in isolated network segments with "Default Deny" inter-VLAN routing.
- **Management Plane Isolation:** Establish a dedicated Out-of-Band (OOB) Management VLAN (VLAN 60) to isolate administrative traffic. This ensures that the switch, and access point configuration interfaces are invisible to the general network and protected from credential sniffing or unauthorized access.
- **Security Stack & Telemetry:** Deploy a centralized security suite on a Raspberry Pi (VLAN 30) utilizing Splunk for log aggregation. Establish a telemetry pipeline using Syslog for network infrastructure, NetFlow/IPFIX for traffic flow analysis, and lightweight log agents (such as Splunk Universal Forwarder) for endpoint visibility. This architecture prioritizes metadata-driven analysis over full packet capture to optimize CPU and storage resources while maintaining the capability for on-demand Deep Packet Inspection (DPI) via temporary port mirroring for advanced troubleshooting.
- **Research Containment:** Create a dedicated Sandbox VLAN (VLAN 50) with strict firewall rules to prevent lateral movement, proxied/ limited WAN access, and a "Strict Kill-Switch" to ensure malware research remains contained.
- **Hardware-Accelerated Gateway:** Deploy a custom-built OPNsense appliance using the HP t620 Plus. This configuration leverages AES-NI for encrypted traffic and an Intel i350-AM2 NIC for multi-gigabit internal throughput and driver stability. These components were hand selected to optimize the balance between security, reliability, speed, and cost-efficiency. This deployment grants comprehensive control over the routing and firewall planes, facilitating the implementation of IPS (Intrusion Prevention System) and NGFW capabilities, while enabling granular filtering of all East-West traffic between isolated VLAN segments.

#### **Professional Governance**
- **Change Management:** Document every infrastructure modification via an "Action Taken" document and a formal Change Request (CR) process. This includes baseline assessments, technical justifications, and rollback plans to simulate enterprise IT operations.
- **Technical Documentation:** Maintain High-Level Logical Topologies (before and after), a comprehensive Network Addressing Plan, and other forms of documentation as the project scales.
- **Validation:** Produce visual evidence of security controls, including screen-recorded "Ping Audits" to verify firewall block rules and a video presentation defending the architecture to a simulated Board of Directors.
